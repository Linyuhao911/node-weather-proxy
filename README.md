# 天气预报查询网页项目学习总结

通过完成这个天气预报查询网页项目，我系统地练习了现代 Web 开发的完整流程，收获了从前端交互设计到后端 API 代理的全栈开发经验，显著提升了实际编程能力与工程思维。

## 前端部分
在前端开发过程中，我主要使用了 HTML、CSS 和 JavaScript 来构建用户界面。页面采用简洁美观的玻璃摩卡风格设计，通过 Flex 布局和媒体查询实现了桌面与移动端的响应式适配；使用 JavaScript 实现了用户交互逻辑，包括输入城市名称、触发查询、动态渲染天气数据（如城市、天气描述、温度、更新时间）以及错误提示处理。这让我深刻体会到用户体验的重要性：一个小小的加载动画、背景图片根据天气动态切换，都能大幅提升页面的吸引力和沉浸感。同时，前端异步请求（POST 方式）和 DOM 操作的实践，也加深了我对事件驱动编程的理解。

## 后端部分
后端部分是我本次项目最大的挑战与收获点。我使用 Node.js 和 Express 框架搭建了一个轻量级代理服务器（server.js），成功实现了对心知天气 API 的安全调用。重点解决了 API 签名验证问题：通过 crypto-js 库生成 HMAC-SHA1 签名，确保请求参数（ts、ttl、uid、sig）的正确性；采用 POST 路由接收前端数据、拼接 API URL 并使用 fetch 获取真实天气信息；同时实现了参数校验、错误处理和静态文件托管功能。这让我第一次真正理解了“前后端分离”和“API 代理”的实际意义——既保护了 API 密钥安全，又避免了前端跨域问题。更重要的是，我学会了如何阅读第三方 API 文档、处理签名认证、异步错误捕获等生产级开发技能。

## 项目成果
项目整体成果令人满意：一个功能完整、可独立运行的天气查询网页，仅需 `node server.js` 即可在本地启动，支持任意城市实时天气查询，界面美观、响应迅速。这个项目不仅巩固了课堂所学的 Web 技术栈，还让我具备了独立开发小型全栈应用的能力，代码已整理并上传至 GitHub 作为个人作品集的一部分。

## 未来改进方向
当然，项目仍存在改进空间：
- 前端可增加多日预报、自动定位、空气质量等功能；
- 后端可引入 dotenv 管理密钥、添加缓存机制和日志记录；
- 整体可支持 PWA 离线访问和部署到 Vercel 实现在线访问。

未来我将继续迭代优化，目标是将其打造成一个功能更丰富、体验更优秀的天气查询工具。

此次项目让我深刻认识到：理论学习固然重要，但真正的能力提升来自于动手实践与问题解决。通过反复调试签名错误、优化界面细节，我培养了耐心、细致和系统化的思维习惯。这段经历将成为我后续学习 Web 开发、甚至全栈方向的宝贵财富。
